# 项目阶段划分

## 1. 文档目的

本文档用于把整个项目按工程目标拆成明确阶段，避免后续开发继续以零散补丁方式推进。

每个阶段只负责一类核心目标，并给出：
- 阶段目标
- 主要产出
- 进入条件
- 完成标准
- 不包含内容

---

## 2. 阶段总览

当前建议将整个项目划分为 6 个阶段：

1. 阶段一：图形运行链路可行性验证
2. 阶段二：运行时稳定性与历史态一致性
3. 阶段三：编辑器工作流与时间轴增强
4. 阶段四：多图层 / 多区域 / 效果系统扩展
5. 阶段五：产品化与长期演进能力
6. 阶段六：可视化编程引擎 + 实时动画 + 专业时间轴 + 渲染导出 + 音频混音 + 视频效果链 + Asset Genome + AI Director + WDL 声明式渲染 DSL（Step 25-37.6）

### 当前状态

### 主要产出

#### Step 25-28：可视化编程引擎 + Material/Shader 系统

- Audio Reactive + Sensor Input（音频/麦克风/MIDI/摄像头驱动视觉反馈）

#### Step 31.x：专业时间轴（Pro Timeline）

- audioMix（混音核心：dB/linear 转换、等功率声像、mute/solo 优先级、效果链操作）

- ProTimelineAudioMixer（UI：轨道列声像旋钮/音量推子/M/S/效果 + 主输出 + 电平表 RAF 驱动）
- 78 测试通过（AM 21 + AE 13 + AC 10 + AQ 10 + AV 6 + AS 18）

#### Step 34：视频效果链 Effect Chain

- effectChain（17 种效果类型 × 5 大类:色彩 / 模糊 / 变换 / 风格化 / 合成）
- 纯函数不可变操作 + 参数验证（clamp 越界值）+ 结构完整性验证
- 5 个内置预设:电影感 / 复古 / 梦幻 / 故障 / 锐利高清
- effectChainStore（Pinia Store 管理 clipId→EffectChain 映射,Map 响应式触发）
- ProTimelineEffectChain（UI:与 selectionStore 联动,按大类分组添加菜单,效果列表 + 参数编辑器）
- 77 测试通过（EC 11 + CH 17 + Q 6 + V 9 + P 9 + M 7 + S 18）

#### Step 35.1：Asset Genome 基础 — Asset Registry Core

- 14 种 AssetKind × 4 大 AssetCategory 类型系统（media / shader / scene / config）
- AssetMetadata + AssetRecord 数据模型（id / kind / category / name / source / version / tags）
- 注册表纯函数 CRUD + 查询 + 验证 + 旧版 Asset 兼容转换
- assetRegistryStore（Pinia Store + 响应式 Map 触发 + computed 查询）
- 75 测试通过（T 8 + F 5 + R 18 + Q 15 + V 9 + L 2 + S 18）

#### Step 35.2：Asset Reference Graph — 依赖关系 DAG + 引用追踪

- ReferenceGraph 双向邻接表（adjacency + reverseIndex）,支持 uses / extends / embeds 三种引用类型
- 工厂 + CRUD + 查询（getReferences / getReferencers / hasReference / byType / degree）
- referenceGraphStore（Pinia Store + 响应式 Map 触发 + computed 查询）
- 38 测试通过（F 6 + C 10 + Q 11 + S 11）

#### Step 35.3：Impact Analysis — 影响分析 + 循环检测 + 拓扑排序

- getDownstreamImpact / getUpstreamDependencies（BFS 传播）
- getDownstreamDepth / getDownstreamImpactDetails（带深度与路径的影响集）
- hasCycle（DFS 三色标记法） / findCycles（环路路径列表） / topologicalSort（Kahn 算法）
- 18 测试通过（IA 10 + IAC 5 + IAD 3）

#### Step 35.4：Content Hash & Dedup — 内容哈希 + 去重 + 相似检测

- fnv1a32（FNV-1a 32-bit 快速哈希）+ computeContentHash（元数据指纹）
- computeBinaryHash（SHA-256 / FNV-1a 降级）
- buildHashIndex / findDuplicates / findDuplicatesOf（基于 contentHash 去重）
- computeSimilarity（Jaccard 名称 0.4 + 标签 0.4 + 种类 0.2） + findSimilarPairs / findSimilarTo
- generateDedupSuggestions（保留最新创建的,返回 DedupSuggestion[]）
- 25 测试通过（CH 9 + CD 6 + CS 7 + CDS 3）

#### Step 35.5：Lazy Loading & Index — 加载状态管理 + 多维索引 + 懒加载协调器

- LoadState 状态机（unloaded / loading / loaded / error） + LoadStatusTable + 状态函数
- AssetIndex 多维倒排索引（name / kind / category / tag） + 查询函数（大小写不敏感 / OR 语义）
- LazyLoader 协调器：requestLoad（入队）+ executeLoad（执行）+ executeBatchLoads（顺序执行避免状态合并冲突）
- 28 测试通过（LS 11 + AI 10 + LL 7）

#### Step 35.6：Asset Browser UI — 资产浏览器面板

- ProTimelineAssetBrowser.vue：模态弹窗设计（搜索框 + 大类筛选 + 标签栏 + 资产列表按大类分组 + 详情面板）
- 详情面板展示：ID / 种类 / 大类 / 来源 / 版本 / 大小 / 时间 + 标签 + 引用数（出边 / 入边） + 描述
- 集成到 ProTimeline.vue,使用 useAssetRegistryStore + useReferenceGraphStore
- --pf-* 设计令牌 + cubic-bezier(0.22, 1, 0.36, 1) 180ms 过渡

#### Step 35.7：Asset Packaging — 打包导出 + 导入

- AssetPackage 接口（formatVersion=1 + name + createdAt + assets[] + references[]）
- createPackage / serializePackage（JSON） / deserializePackage + 格式校验
- validatePackage（包名非空 + 引用 ID 存在性 + 资产 ID 唯一性）
- mergePackage（已存在 ID 跳过,返回新集合 + ImportResult）
- createPackageWithIds（过滤导出,只保留 source+target 都在列表中的引用）
- 18 测试通过（AP01-AP18）

#### Step 36.1：Director Context Engine — 上下文序列化

- DirectorContext 接口（summary / canvasSize / layerCount / effectCount / hasTimeline / modifiableParams）
- ModifiableParam 接口（targetEntity / targetId / paramKey / currentValue / description）
- 参数描述映射（OPCODE_DESCRIPTION / LAYER_PARAM_DESCRIPTION / EFFECT_PARAM_DESCRIPTION）
- buildDirectorContext / buildCreateModeContext / buildModifyModeContext 三模式上下文构建
- serializeLayer 输出格式：`[0] id=layer_01 SOLID_COLOR(纯色填充) color=[1,0,0,1] blendMode=normal`
- 18 测试通过（DC01-DC13 + CC01-CC02 + MC01-MC03）

#### Step 36.2：Enhanced Intent + System Prompt — 增强意图解析 + 上下文感知

- DirectorMode = 'create' | 'modify' | 'animate' | 'analyze' 四种操作模式
- EnhancedIntent extends DirectorIntent（添加 mode / keywords / referencedLayerIds）
- 关键词数组（中英文）：CREATE_KEYWORDS / MODIFY_KEYWORDS / ANIMATE_KEYWORDS / ANALYZE_KEYWORDS
- parseEnhancedIntent（模式检测 + 关键词提取 + 图层引用检测：直接 ID + 序数引用"第一个图层"）
- parseChineseNumber（解析中文数字：一/二/三/...）
- buildContextAwareSystemPrompt（按模式注入不同上下文）
- decideWithContext（增强版决策,注入上下文到系统提示词）
- convertLLMOutputToPatchesWithContext（modify 模式使用引用图层 ID,animate 模式跳过动画字段）
- 24 测试通过（EI01-EI12 + SP01-SP06 + DC01-DC06）

#### Step 36.3 + 36.4：Multi-turn Conversation + Timeline Auto-Generation

- ConversationRole = 'user' | 'director' / ConversationMessage / ConversationSession
- 会话管理（createConversation / addUserMessage / addDirectorMessage / clearConversation）
- serializeConversation（序列化历史为文本：`[用户] ...` / `[Director] ...` 格式）
- buildConversationSystemPrompt（包含历史 + 上下文的系统提示词）
- extractAnimationParams（从 LLM 输出提取 animateFrom / animateTo / duration）
- createTrackFromAnimation（生成 2 个关键帧的 TimelineTrack）
- createTimelineFromAnimations（生成完整 TimelineContent,取最大 duration 为总时长）
- generateTimelineFromLLM（一步到位从 LLM 输出生成 Timeline）
- converse 多轮对话主入口（集成意图解析 + 上下文提示词 + LLM 调用 + Timeline 生成）
- 21 测试通过（S01-S05 + H01-H04 + C01-C04 + T01-T08）

#### Step 36.5：Director Panel UI — AI Director 对话式面板

- ProTimelineDirectorPanel.vue（模态面板,按钮触发 + 右侧固定定位）
