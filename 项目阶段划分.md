# 项目阶段划分

## 1. 文档目的

本文档用于把整个项目按工程目标拆成明确阶段，避免后续开发继续以零散补丁方式推进。

每个阶段只负责一类核心目标，并给出：
- 阶段目标
- 主要产出
- 进入条件
- 完成标准
- 不包含内容

---

## 2. 阶段总览

当前建议将整个项目划分为 6 个阶段：

1. 阶段一：图形运行链路可行性验证
2. 阶段二：运行时稳定性与历史态一致性
3. 阶段三：编辑器工作流与时间轴增强
4. 阶段四：多图层 / 多区域 / 效果系统扩展
5. 阶段五：产品化与长期演进能力
6. 阶段六：可视化编程引擎 + 实时动画 + 专业时间轴 + 渲染导出 + 音频混音 + 视频效果链 + Asset Genome（Step 25-35.1）

## 4. 阶段二：运行时稳定性与历史态一致性

### 阶段目标

### 当前状态

- **已完成并可封板**

### 当前完成说明

阶段二当前已经完成：
- 运行时错误第一版分层
- 回放错误第一版分层
- WebGPU 初始化失败路径测试
- 历史态正式投影模块化
- 检查面板历史态投影复查
- 渲染签名一致性验证
- 结果级一致性数据进入帧数据面板

### 不再纳入阶段二的内容

以下内容从现在开始不再视为阶段二未完成项：
- 更细错误枚举进一步扩展
- 像素级一致性验证

- 三层 Cache Key 细化（StaticKey 含 blendModes / StructuralKey / DynamicKey）
- 工件迁移 descriptorBuffer V1→V2
- 203 项自动化测试全部通过
- 文档对照审计完成

## 8. 阶段六：可视化编程引擎 + 实时动画 + 专业时间轴 + 渲染导出

### 阶段目标

从产品化阶段进入可视化编程引擎阶段,把 PixelForge 从"AI 生成器"升级为"可视化编程 + 实时动画 + GPU 计算 + 专业时间轴 + 渲染导出 + 音频混音 + 视频效果链"的完整视觉引擎。对应原 Phase F（Timeline / Revision）的实际落地与超越。

### 当前状态

- **进行中**（Step 25-34 已完成,Step 35+ 待规划）

### 主要产出

#### Step 25-28：可视化编程引擎 + Material/Shader 系统
- RenderIR Graph Editor（节点化编辑,DAG 图结构）
- Graph Runtime（拓扑排序 + 内容寻址缓存 + 纹理池）
- Graph Editor 高级交互（无限画布 + 节点拖拽/连线 + 小地图 + 快捷键）
- Material/Shader Node System（WGSL 自动生成 + Shader Cache + GPU 核融合）

#### Step 29-30：实时动画 + 输入系统
- Advanced Timeline + Animation Engine（关键帧 + 曲线 + ParamPatch + GPU Uniform 更新）
- Audio Reactive + Sensor Input（音频/麦克风/MIDI/摄像头驱动视觉反馈）
- 动画执行链路打通：Timeline → evaluateTrack → ParamPatch → GraphNode/MaterialNode → compiler → WGSL + UniformBuffer → WebGPU → Canvas

#### Step 31.x：专业时间轴（Pro Timeline）
- Step 31.1：Timeline Core（Project/Sequence/Track/Clip 数据模型 + Command 模式 + Snap 吸附）
- Step 31.2：Clip CRUD UI（ProTimeline 完整 UI + 拖拽 + 右键菜单）
- Step 31.3：多轨道编辑增强（重命名/着色/删除/复制/重排序）
- Step 31.4：高级片段编辑（剪切/分割/波纹删除/多选）
- Step 31.5：Frame Scheduler（60FPS 时间预算 + 5 阶段调度链 + GPU 资源生命周期）
- Step 31.6：多 Sequence 切换 + 嵌套 Sequence（Project/SequenceCommands/NestedSequenceResolver）
- Step 31.7：跨 Sequence 标尺对齐 + 拖拽片段
- Step 31.8：嵌套 Sequence 双击进入子 Sequence 编辑模式（面包屑导航）
- Step 31.9：Sequence 模板/预设库（6 内置 + localStorage 自定义）

#### Step 32：渲染导出模块
- RenderConfig（配置 + 6 预设 + 验证 + 帧序列计算）
- RenderPipeline（状态机 + 逐帧调度 + 暂停/取消/失败处理）
- RenderStore（Pinia Store + 进度 computed + 控制 actions）
- ProTimelineRenderPanel（配置编辑 UI + 预设 + 进度条 + 输出列表）

#### Step 33：音频混音器

- audioMix（混音核心：dB/linear 转换、等功率声像、mute/solo 优先级、效果链操作）
- audioMixerStore（Pinia Store + Web Audio API 集成：AudioContext/AnalyserNode/电平表）
- ProTimelineAudioMixer（UI：轨道列声像旋钮/音量推子/M/S/效果 + 主输出 + 电平表 RAF 驱动）
- 78 测试通过（AM 21 + AE 13 + AC 10 + AQ 10 + AV 6 + AS 18）

#### Step 34：视频效果链 Effect Chain

- effectChain（17 种效果类型 × 5 大类:色彩 / 模糊 / 变换 / 风格化 / 合成）
- 纯函数不可变操作 + 参数验证（clamp 越界值）+ 结构完整性验证
- 5 个内置预设:电影感 / 复古 / 梦幻 / 故障 / 锐利高清
- effectChainStore（Pinia Store 管理 clipId→EffectChain 映射,Map 响应式触发）
- ProTimelineEffectChain（UI:与 selectionStore 联动,按大类分组添加菜单,效果列表 + 参数编辑器）
- 77 测试通过（EC 11 + CH 17 + Q 6 + V 9 + P 9 + M 7 + S 18）

#### Step 35.1：Asset Genome 基础 — Asset Registry Core

- 14 种 AssetKind × 4 大 AssetCategory 类型系统（media / shader / scene / config）
- AssetMetadata + AssetRecord 数据模型（id / kind / category / name / source / version / tags）
- 注册表纯函数 CRUD + 查询 + 验证 + 旧版 Asset 兼容转换
- assetRegistryStore（Pinia Store + 响应式 Map 触发 + computed 查询）
- 75 测试通过（T 8 + F 5 + R 18 + Q 15 + V 9 + L 2 + S 18）

### 完成标准

- 2314 项自动化测试全部通过（对比阶段五封板时 258 项,增长 9.0 倍）
- vue-tsc 零新增 TS 错误（7 个预存错误均为阶段五之前遗留）
- 可视化编程引擎、动画系统、输入系统、专业时间轴、渲染导出、音频混音、视频效果链、Asset Genome 基础 全链路打通
