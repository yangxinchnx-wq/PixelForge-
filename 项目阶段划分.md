# 项目阶段划分

## 1. 文档目的

本文档用于把整个项目按工程目标拆成明确阶段，避免后续开发继续以零散补丁方式推进。

每个阶段只负责一类核心目标，并给出：
- 阶段目标
- 主要产出
- 进入条件
- 完成标准
- 不包含内容

---

## 2. 阶段总览

当前建议将整个项目划分为 6 个阶段：

1. 阶段一：图形运行链路可行性验证
2. 阶段二：运行时稳定性与历史态一致性
3. 阶段三：编辑器工作流与时间轴增强
4. 阶段四：多图层 / 多区域 / 效果系统扩展
5. 阶段五：产品化与长期演进能力
6. 阶段六：可视化编程引擎 + 实时动画 + 专业时间轴 + 渲染导出 + 音频混音 + 视频效果链 + Asset Genome + AI Director + WDL 声明式渲染 DSL（Step 25-37.6）
