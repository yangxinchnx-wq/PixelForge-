# 项目阶段划分

## 1. 文档目的

本文档用于把整个项目按工程目标拆成明确阶段，避免后续开发继续以零散补丁方式推进。

每个阶段只负责一类核心目标，并给出：
- 阶段目标
- 主要产出
- 进入条件
- 完成标准
- 不包含内容

---

## 2. 阶段总览

当前建议将整个项目划分为 5 个阶段：

1. 阶段一：图形运行链路可行性验证
2. 阶段二：运行时稳定性与历史态一致性
3. 阶段三：编辑器工作流与时间轴增强
4. 阶段四：多图层 / 多区域 / 效果系统扩展
5. 阶段五：产品化与长期演进能力

## 4. 阶段二：运行时稳定性与历史态一致性

### 阶段目标

在阶段一的可行性基础上，提升运行时稳定性，并让历史态在代码、状态和界面层都形成一致闭环。

### 当前状态

- **已完成并可封板**

### 当前完成说明

阶段二当前已经完成：
- 运行时错误第一版分层
- 回放错误第一版分层
- WebGPU 初始化失败路径测试
- 历史态正式投影模块化
- 检查面板历史态投影复查
- 渲染签名一致性验证
- 结果级一致性数据进入帧数据面板

### 不再纳入阶段二的内容

以下内容从现在开始不再视为阶段二未完成项：

## 6. 阶段四：多图层 / 多区域 / 效果系统扩展

### 阶段目标

将单图层渲染扩展为多图层混合渲染，引入区域边界和静态效果后处理管线。

### 当前状态

- **已完成并可封板**

### 主要产出

- Render IR V2 扩展支持多图层、区域边界、静态效果
- WGSL 多图层循环求值与 6 种混合模式（Normal/Multiply/Screen/Overlay/Add/Subtract）
- 独立效果后处理 Pass（effect_post.wgsl）
- 三层 Cache Key 细化（StaticKey 含 blendModes / StructuralKey / DynamicKey）
- 工件迁移 descriptorBuffer V1→V2
- 203 项自动化测试全部通过
- 文档对照审计完成

### 完成说明

详见 `阶段四完成定义.md`。
