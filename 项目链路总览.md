# PixelForge 项目链路总览

## 1. 项目定位

PixelForge 是一个可视化编程视觉引擎,从"AI 生成器"演进为完整的视觉创作平台。

当前阶段(阶段六)已具备以下能力:
- 可视化节点图编程 + Material/Shader 节点系统
- 实时动画引擎 + 音频/MIDI/摄像头/传感器输入驱动
- 专业时间轴(Pro Timeline,多 Sequence + 嵌套 + 模板库)
- 渲染导出 + 音频混音器 + 视频效果链
- Asset Genome(资产注册表 + 依赖图 + 影响分析 + 去重 + 懒加载 + 浏览器 UI + 打包)
- AI Director(对话式创作 + Timeline 自动生成)
- WDL 声明式渲染 DSL(Monaco 编辑器 + 语法高亮 + 自动补全 + 错误内联 + Graph 双向同步)
- 渲染性能优化(Profiler + GPU 资源池化 + 多 Pass 管线 + 三级签名缓存)
- 产品化(设置面板 + 快捷键 + 项目导入导出 + 错误处理统一化)

## 3. 核心链路

### 3.1 渲染主链路

```
自然语言 prompt / WDL 文本 / 节点图编辑
  → RequirementClarifier / WDL Parser / GraphEditor
  → RenderIR(layers / effects / regions)
  → Compiler(RegionCompiler / GraphCompiler / MaterialCompiler)
  → WGSL 着色器 + UniformBuffer
  → WebGPU(计算着色器并行求值)
  → Canvas 呈现
  → FrameRecord 记录(支持回放与签名校验)
```

### 3.2 动画执行链路(Step 29-30)

```
Timeline(step)
  → evaluateTrack(关键帧 + 曲线插值)
  → ParamPatch(参数补丁)
  → GraphNode / MaterialNode(节点图更新)
  → compiler(WGSL 重新生成)
  → UniformBuffer(GPU 更新)
  → WebGPU → Canvas(实时渲染)

InputDriver(音频/MIDI/摄像头/传感器)
  → FeatureExtractor(特征提取)
  → InputRouter(signalId → targetKind/nodeId/property)
  → ParamPatch(信号驱动参数更新)
  → WebGPU → Canvas(实时反馈)
```

### 3.3 专业时间轴链路(Step 31.x)

```
Project → Sequence → Track → Clip(bigint 微秒精度)
  → Command 模式(undo/redo)
  → Snap 吸附
  → FrameScheduler(60FPS 时间预算 + 5 阶段调度)
  → GPU 资源生命周期管理
  → 嵌套 Sequence 解析(NestedSequenceResolver)
  → 模板库(6 内置 + localStorage 自定义)
```

### 3.4 渲染导出链路(Step 32)

```
RenderConfig(预设 + 帧序列计算)
  → RenderPipeline(状态机:逐帧调度)
  → 每帧:渲染 → 截图 → 编码
  → 暂停/取消/失败处理
  → 输出文件列表
```

### 3.5 WDSL 链路(Step 37-38)

```
WDL 文本
  → Lexer(词法分析)
  → Parser(递归下降,AST)
  → Compiler(AST → RenderIR)
  → Validator(语义校验)
  → runtimeStore.setRenderIR
  → 渲染主链路

Monaco Editor:
  wdlMonarch(语法高亮) + wdlCompletion(7 类补全) + wdlDiagnostics(错误内联)
  + wdlGraphSync(WDL ↔ Graph 双向同步) + wdlTemplates(9 场景模板)
  + wdlTimelineBinding(7 个 Clip 属性绑定)
```

### 3.6 错误处理链路(Step 40.4)

```
未知错误(throw / console.error / window.alert)
  → errorStore.push(caught, userHint?)
  → classifyError(caught)  [shared/errors.ts]
  → RuntimeErrorInfo { code, severity, source, recoverable }
  → ErrorEntry(LRU 50 队列 + activeError)
  → ErrorToast.vue(右上角通知)

组件级错误:
  ErrorBoundary(onErrorCaptured) → fallback UI + retry

全局兜底:
  app.config.errorHandler(main.ts) → errorStore.push
```

## 4. 当前目录职责

### `src/shared`
跨层共享类型与基础值层公约(errors.ts 错误工厂 + ids.ts + seed.ts)。

### `src/compiler`
编译相关能力:Render IR → 区域渲染工件 + Patch 协议 + Compile Cache。

### `src/runtime`
运行时图形能力:WebGPU 初始化 + 渲染 + 呈现 + 渲染签名 + FrameScheduler + GPU 资源管理 + 多 Pass 管线 + 渲染缓存 + Profiler。

### `src/services/frame`
帧数据服务层:统一帧记录接口 + IndexedDB 仓储 + 适配与播放控制。

### `src/stores`
前端状态层:runtime / timeline / history / errorStore / projectStore 等。

### `src/components`
前端界面:editor(graph/timeline/pro-timeline/CommandPalette/ErrorToast/ErrorBoundary 等)+ ui。

### `src/graph`
可视化编程引擎:Graph Editor + Graph Runtime(拓扑排序 + 内容寻址缓存 + 纹理池)。

### `src/material`
Material/Shader Node System:WGSL 自动生成 + Shader Cache + GPU 核融合。

### `src/animation`
动画引擎:关键帧 + 曲线 + ParamPatch + UniformUpdater + InputDriver。

### `src/input`
输入系统:audio(FFT/节拍检测)+ midi + camera(运动检测)+ sensor + inputRouter。

### `src/editor`
编辑器模块:asset-genome(注册表/引用图/影响分析/去重/懒加载/打包)+ audio(混音器)+ effects(效果链)+ render(导出)+ timeline(Pro Timeline 核心)。

### `src/world`
L3 大规模工程:director(AI Director)+ orchestrator(L3 编排)+ revision(修订层)+ sceneGraph(场景图)+ timeline(世界时间轴)+ wdl(WDL DSL 全链路)。

### `src/preferences`
用户偏好:settingsStore(主题/画布/自动保存/性能监控)+ theme。

### `src/project`
项目管理:serializer + fileSystem + autosave + recentProjects + projectValidator + projectExport + projectStore。

### `src/composables`
组合式函数:commandRegistry + useCommandShortcuts。

## 6. 帧记录与回放

### 6.1 帧记录

无论成功还是失败,每次渲染都写入帧记录:
- `RuntimeFrameRecord`:完整运行时状态快照
- `FrameSnapshot`:界面层统一消费格式
- IndexedDB 持久化(阶段五)

### 6.2 回放

- `replayFrame(frame)`:历史帧回放
- 渲染签名校验(签名一致性验证)
- 错误处理:`replay/*` 错误码(4 码)

### 6.3 一致性验证

- 像素级回放一致性(阶段五)
- 渲染签名三级缓存(Step 39.4)

## 7. 设计原则

### 7.1 界面层统一消费 FrameSnapshot

界面层不直接依赖运行时内部结构。所有数据先适配成 `FrameSnapshot` 再进入界面层。

### 7.2 错误归一化

所有错误经 `classifyError()` 归一化为 `RuntimeErrorInfo`,再经 `severityToLevel()` 转换为 `NotificationLevel`,统一由 `errorStore` + `ErrorToast` 展示。

### 7.3 声明式渲染

WDL DSL 允许用户通过文本描述场景,替代手动节点连接。WDL ↔ Graph 双向同步保证一致性。

### 7.4 GPU 资源复用

TexturePool + BufferPool 统一管理 GPU 资源,size 分桶避免碎片化,LRU 淘汰策略。
