# 阶段二运行时错误分层说明

## 1. 文档目的

本文档专门定义阶段二引入的运行时错误分层与回放错误分层，避免后续继续依赖松散字符串做判断。

本文档只回答：
- 当前有哪些错误类型
- 这些错误在哪一层产生
- 界面层应该如何消费它们

---

## 2. 错误分层结构

当前项目已经明确分成两类错误：

### 2.1 运行时错误

运行时错误主要发生在：
- WebGPU 初始化
- GPU 适配器获取
- GPU 设备创建
- 画布上下文创建

对应类型：
- `RuntimeErrorInfo`
- `RuntimeErrorCode`

当前支持的错误码：
- `runtime/webgpu-unavailable`
- `runtime/adapter-unavailable`
- `runtime/context-unavailable`
- `runtime/device-request-failed`
- `runtime/unknown`

### 2.2 回放错误

回放错误主要发生在：
- 运行时不可用
- 回放数据缺失
- 工件版本不兼容
- 渲染签名不一致

对应类型：
- `ReplayErrorInfo`
- `ReplayErrorCode`

当前支持的错误码：
- `replay/runtime-unavailable`
- `replay/missing-data`
- `replay/incompatible-artifact-version`
- `replay/signature-mismatch`

---

## 3. 当前状态字段

在 `runtimeStore` 中，错误相关状态已经明确拆成：

### 3.1 通用运行时错误状态
- `error: string | null`
- `runtimeError: RuntimeErrorInfo | null`

说明：
- `error` 用于界面层直接展示文本
- `runtimeError` 用于程序化判断与测试

### 3.2 回放错误状态
- `replayStatus: 'idle' | 'success' | 'error'`
- `replayError: string | null`
- `replayErrorInfo: ReplayErrorInfo | null`

说明：
- `replayStatus` 用于界面展示当前回放状态
- `replayError` 用于界面展示当前回放错误文本
- `replayErrorInfo` 用于程序化判断与测试

---

## 4. 当前错误与界面层的映射规则

### 4.1 初始化失败

来源：`initRuntime()`

映射：
- `status = 'error'`
- `error = runtimeError.message`
- `runtimeError = 结构化错误对象`

### 4.2 回放失败

来源：`replayFrame(frame)`

映射：
- `replayStatus = 'error'`
- `replayError = replayErrorInfo.message`
- `replayErrorInfo = 结构化错误对象`

### 4.3 回放成功

映射：
- `replayStatus = 'success'`
- `replayError = null`
- `replayErrorInfo = null`

---

## 5. 当前边界

当前错误分层已经具备：
- 类型定义
- store 状态入口
- 初始化失败路径测试
- 回放错误路径测试

但当前还没有做：
- 更细粒度 GPU 执行错误分类
- 更完整的错误展示策略
- 全局统一错误组件

---

## 6. 当前结论

阶段二的运行时错误分层已经完成第一版落地。

这意味着：
- 错误不再只是文本
- 运行时和回放错误已经分层
- 测试可以直接基于错误码断言

后续如果继续细化，应在此基础上扩展，而不是推倒重来。
